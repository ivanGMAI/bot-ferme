
# 🤖 Bot Ferme Backend

RESTful-сервис на Python для управления пулом тестовых пользователей. Предназначен для автоматизации E2E-тестирования: сервис выдает реальные учетные данные пользователей для эмуляции работы реальных людей на проверяемых платформах.

## 🚀 Функционал
* **User Management:** Создание и хранение учетных данных пользователей в PostgreSQL.
* **E2E Integration:** Механизм блокировки (lock) пользователя на время проведения теста.
* **List API:** Получение списка всех существующих пользователей ботофермы.
* **Cloud Native:** Полная готовность к работе в Kubernetes с поддержкой автоскейлинга (HPA).

## 🛠 Стек технологий
* **Backend:** Python 3.13, FastAPI, SQLAlchemy 2.0 (Async), Alembic.
* **Infra:** Docker, Kubernetes (Minikube), Helm.
* **DevOps:** Makefile, `uv`.

## ⚡️ Быстрый старт (Automation)

Все ключевые операции по развертыванию автоматизированы через **Makefile**.

1. **Подготовьте окружение:**
   ```bash
   minikube start
   

2. **Разверните проект одной командой:**
   ```bash
   make install
   ```

3. **Запустите доступ к API:**
   ```bash
   make up
   ```
   Swagger UI будет доступен по адресу: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

## 🔍 Управление и диагностика

| Команда | Назначение |
| :--- | :--- |
| `make status` | Состояние подов, сервисов и метрики HPA |
| `make logs` | Просмотр логов бэкенда в реальном времени |
| `make test` | Запуск тестов внутри кластера через Helm Test |
| `make top` | Мониторинг потребления ресурсов (CPU/RAM) |
| `make stop` | Принудительная очистка порта 8000 |
| `make uninstall` | Полное удаление приложения из кластера |

## 🧪 Локальное тестирование
Для запуска тестов вне кластера:
```bash
cd src/backend
